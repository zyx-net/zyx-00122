# 离线巡检 Web 应用

适配手机浏览器的离线巡检应用，支持巡检员按点位填写检查项、自动保存草稿、异常上报，管理员配置模板、审核退回返工。所有数据存储在浏览器本地（IndexedDB），离线可用，刷新/重启后数据可恢复。

## 技术栈

- **前端**: React 18 + TypeScript + Vite
- **样式**: TailwindCSS 3
- **状态管理**: Zustand
- **本地存储**: IndexedDB (Dexie.js)
- **路由**: React Router DOM v7
- **图标**: Lucide React

## 功能模块

### 巡检员端
- 🔐 **任务领取**: 浏览可领取任务，一键领取
- 📝 **巡检填写**: 按点位逐步填写检查项（文本/数值/单选/附件）
- 💾 **自动草稿**: 每次修改自动保存到 IndexedDB，刷新页面可恢复
- ⚠️ **异常上报**: 对任意检查项标记异常，填写描述和附件占位
- ✅ **提交校验**: 自动检查必填项、数值范围、模板版本

### 管理端
- 📋 **模板配置**: 创建/编辑巡检模板，定义点位及检查项
- 📦 **任务分配**: 从模板快速创建任务
- 🔍 **审核提交**: 查看巡检结果，通过或退回返工
- 🔄 **退回返工**: 填写返工原因，保留旧答案和历史记录
- 📊 **数据导出**: 导出模板/任务/草稿/提交/异常/日志为 JSON

### 公共模块
- 📜 **事件日志**: 记录所有操作（领取/草稿/提交/退回/异常/拒绝），支持按类型筛选
- 💾 **数据导出**: 选择数据范围，导出 JSON 文件

## 核心校验规则

| 校验类型 | 处理方式 |
|---------|---------|
| 缺必填项 | 阻止提交，高亮第一个错误字段，显示具体错误信息 |
| 数值越界 | 阻止提交，提示"值 X 小于/大于最小/最大值 Y" |
| 旧模板草稿 | 阻止提交，提示模板已更新，请重新填写 |
| 重复提交 | 阻止提交，按钮显示"已提交"禁用状态 |
| 状态覆盖保护 | 更新时比较 updatedAt 时间戳，拒绝覆盖更新的状态 |

## 数据一致性保证

- **草稿时间**: 每次保存更新 `savedAt` 时间戳，显示在巡检页右上角
- **模板版本**: 草稿记录 `templateVersion`，提交时与当前模板比对
- **返工记录**: 每次退回创建独立 Submission 记录，version 递增，保留旧答案和返工原因
- **异常列表**: 按时间倒序展示，记录检查项、描述、附件占位、上报时间
- **事件日志**: 所有操作不可删除，导出的日志与页面展示一致
- **IndexedDB 持久化**: 所有数据存储在浏览器本地数据库，刷新/重启服务后完整保留

## 手机视口验收标准

### 视口配置
- **基准宽度**: 375px（iPhone SE/Android 常见尺寸）
- **viewport meta**: `width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no`
- **最小点击区域**: 44px × 44px
- **输入框高度**: 48px
- **安全区域适配**: 使用 `dvh` 单位，支持 iOS 刘海屏

### 布局验收（375px 宽度验证）

1. **首页**
   - [ ] 两个角色卡片垂直居中，间距合适
   - [ ] 卡片宽度不超过屏幕宽度（max-w-sm 生效）
   - [ ] 点击区域 ≥ 44px，无横向滚动

2. **任务列表**

3. **巡检填写页**
   - [ ] 顶部标题栏显示任务名、返回按钮、草稿时间
   - [ ] 点位名称和进度条在可视区域内
   - [ ] 检查项表单卡片完整显示，输入框可正常点击
   - [ ] 底部点位导航可横向滚动，不被提交按钮遮挡
   - [ ] 提交按钮固定在底部，不遮挡表单内容

   - [ ] 异常卡片完整展示描述、附件、时间

5. **模板配置页**

   - [ ] 底部弹窗宽度 ≤ 375px，不超出屏幕

   - [ ] 检查结果按点位分组展示，层级清晰

   - [ ] 时间轴左侧圆点对齐，连接线完整

### 触控交互验收

## 首次运行

应用首次启动时会自动初始化种子数据：
- 2 个巡检模板（消防设施日常巡检、电梯安全巡检）
- 3 个任务（2 个待领取、1 个进行中）

## 验收流程建议

1. **巡检员流程**: 领取任务 → 填写部分检查项 → 刷新页面 → 验证草稿已恢复 → 故意缺必填项提交 → 验证被阻止 → 补齐后提交 → 查看事件日志
2. **管理员流程**: 新建模板 → 添加点位和检查项 → 从模板创建任务 → 审核提交 → 退回并填写原因 → 验证巡检员端看到返工原因 → 重新提交后审核通过
3. **数据一致性**: 刷新页面、重启开发服务器 → 验证草稿时间、模板版本、返工记录、异常列表、事件日志、导出数据均保持一致
