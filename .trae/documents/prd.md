## 1. 产品概述

离线巡检 Web 应用——为巡检员和管理员提供适配手机浏览器的巡检全流程工具。核心解决无网络环境下按点位检查、草稿断点续填、异常上报、退回返工、数据一致性等问题，所有数据存储在浏览器本地（IndexedDB），重启服务或刷新页面后均可恢复。

- 目标用户：工厂/园区巡检员（现场使用）、巡检管理员（后台配置与审核）
- 核心价值：离线可用、数据不丢失、操作可追溯、冲突可检测

## 2. 核心功能

### 2.1 用户角色

| 角色 | 进入方式 | 核心权限 |
|------|----------|----------|
| 巡检员 | 首页选择"巡检员"身份 | 领取任务、填写草稿、提交巡检、上报异常、查看日志 |
| 管理员 | 首页选择"管理员"身份 | 创建/编辑模板、分配任务、审核提交、退回返工、导出数据 |

### 2.2 功能模块

1. **模板配置页**：创建巡检模板，定义点位及检查项（文本/数值/选择/附件占位），支持版本管理
2. **任务列表页**：展示可领取任务和已领取任务，支持领取操作
3. **巡检填写页**：按点位逐步填写检查项，自动保存草稿到 IndexedDB
4. **草稿管理**：刷新页面自动恢复未提交草稿，显示草稿保存时间
5. **异常上报页**：对任意检查项标记异常并填写异常描述
6. **审核退回页**：管理员审核已提交任务，可退回并填写返工原因
7. **事件日志页**：记录所有操作日志（领取/保存/提交/退回/异常），支持按时间和类型筛选
8. **导出页**：导出日志、草稿、任务数据为 JSON 文件

## 3. 核心流程

### 3.1 巡检员流程
巡检员登录 → 浏览可领取任务 → 领取任务 → 按点位填写检查项 → 自动保存草稿 → 标记异常 → 提交（校验通过）→ 任务状态变更

### 3.2 管理员流程
管理员登录 → 创建/编辑模板 → 分配任务 → 审核提交 → 退回返工（附原因）→ 巡检员修改后再次提交 → 审核通过

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深蓝 (#1E3A5F) + 琥珀色 (#F59E0B) 作为强调色，传达专业与警觉
- **背景**：浅灰 (#F3F4F6) 配白色卡片，层次分明
- **按钮**：圆角 (rounded-lg)，主操作用琥珀色填充，次要操作用描边
- **字体**：Noto Sans SC（中文优先），数字用 Tabular Nums 等宽
- **布局**：移动端优先，底部固定操作栏，顶部精简导航
- **图标**：Lucide 图标库，线条风格

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 首页 | 角色选择 | 居中两个大卡片，图标+文字，点击切换高亮 |
| 模板配置页 | 模板列表 | 卡片列表，右上角"+"按钮新建 |
| 模板配置页 | 模板编辑器 | 手风琴展开点位，每个检查项为可拖拽卡片 |
| 任务列表页 | 任务卡片 | 状态标签（彩色圆点）、点位进度、领取按钮 |
| 巡检填写页 | 点位导航 | 底部横向滚动的点位圆点，已完成为实心 |
| 巡检填写页 | 检查项表单 | 分组卡片，输入框/滑块/选择器/附件占位符 |
| 审核页 | 提交详情 | 时间轴展示提交历史，高亮变更项 |
| 事件日志页 | 日志列表 | 紧凑时间轴，类型图标+时间戳+摘要 |
| 导出页 | 导出面板 | 勾选数据范围，底部"导出"按钮 |

### 4.3 响应式设计

- **移动优先**：所有页面以 375px 宽度为基准设计
- **触控优化**：最小点击区域 44px，输入框高度 48px
- **单手操作**：核心操作在屏幕下半部分
- **视口设置**：`<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1, user-scalable=no">`
